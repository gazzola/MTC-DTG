 É possível trabalhar com grande qantidade de dados, manter o
histórico de atualizações sobre eles e consultar estados passados dos dados de
maneira eficiente. Como trabalhos futuros pretende-se implementar as funcionalidades temporais
diretamente no código fonte de um banco de dados NoSQL ao invés de fazer um
"Proxy". Essa implementação trará mais ganho de performance para as consultas. = Referências =
